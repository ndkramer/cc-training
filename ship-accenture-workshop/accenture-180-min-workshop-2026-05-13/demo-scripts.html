Before the session
Preflight for ALL demos

Every block below is select-all on click. Run them in order from order-service/. Stop and fix anything that doesn't print the expected line — don't go on stage with a yellow check.

Step 1 — cd into the demo repo

cd "Accenture 180 Min Workshop 2026-04-15/order-service" pwd # → .../Accenture 180 Min Workshop 2026-04-15/order-service

Step 2 — kill anything still listening on port 3001

A prior npm run mcp:http often leaves a zombie node process bound to 3001. A new server will silently fail to bind and /mcp will show a red row.

npm run mcp:kill # → ports cleared: 3001

Step 3 — install deps (first run on a fresh clone only)

npm install # → pulls @modelcontextprotocol/sdk. Skip if node_modules/ already exists.

Step 4 — reset the demo state

npm run reset-demo # → re-plants the Math.ceil bug in lib/pricing.js, restores CLAUDE.md, resets data. # (./reset-demo.sh and reset-demo.cmd are thin shims — same thing.)

Step 5 — confirm the planted bug is back

npm test # → expect: 3 pass, 1 FAIL in test/pricing.test.js # "cancellation on day 15 of 30 bills for 15 days"
Expected: npm test before reset
✖ cancellation on day 15 of 30 bills for 15 days (~2ms) AssertionError [ERR_ASSERTION]: Expected values to be strictly equal: 16 !== 15 ✔ full 30-day cycle bills for 30 days ✔ same-day cancellation bills for 0 days ✔ prorated amount for 15 of 30 days at $9000/mo ℹ tests 4 ℹ pass 3 ℹ fail 1
Two-terminal model (required for Demo 6):
Terminal 1 — stays on npm run mcp:http the entire session. Never close it.
Terminal 2 — this is where you run claude. Both terminals must be in order-service/.

Step 6 — boot the order-store MCP server in Terminal 1 (leave running)

The order-store MCP serves live order data over localhost HTTP. Demo 6 uses it for the "Which orders have drift over 5%?" prompt. No external creds needed — it reads lib/orders.js in-process.

npm run mcp:http # → order-store listening on http://localhost:3001/mcp

Leave Terminal 1 here — do the rest in Terminal 2.

Step 7 — verify the port is bound (Terminal 2)

npm run mcp:check # → port 3001: LISTENING # If it shows "free", scroll Terminal 1 for the error.

Step 8 — launch Claude Code from order-service/ (Terminal 2)

The directory matters. Claude Code reads .mcp.json from the working directory on startup — that file registers the order-store MCP. If you launch Claude from any other directory, /mcp will show nothing.

pwd # must print .../Accenture 180 Min Workshop 2026-04-15/order-service claude

First time on C4E? Run claude auth login once before this step.

Step 9 — confirm the MCP is connected (inside Claude)

/clear /mcp

Expect order-store to show connected.
If red: the server didn't bind. Switch to Terminal 1 — look for an EADDRINUSE error. Re-run Step 2 (kill ports), then Step 6 (restart server), then /mcp again.
If missing entirely: Claude wasn't launched from order-service/ — exit, cd back, and re-run Step 8.

Step 10 — room hygiene

  • Terminal font size ≥ 18pt, light theme if possible
  • Close Slack, silence notifications, Do Not Disturb on
  • Send one throwaway prompt in Claude so the session is warm
  • Browser tab open to demo-scripts.html on a second monitor (this page) — every prompt is select-all-on-click
  • Demo 7 needs tmux — start your main session inside tmux new -s workshop from the beginning
Re-running between sessions: Steps 2 → 4 → 5 → 6 → 9. Skip 1, 3, 8, 10 if you didn't close anything.

Demo 5
Wire up MCP — localhost HTTP transport
~90s · shown at ≥2.0h · needs two terminals
Goal:

Wire the order-store MCP server in two commands. Then ask a natural question — Claude calls the MCP tool directly. No grep, no file read. A structured tool call to your API, visible in the transcript.

Pre-flight

  • cd order-service && npm install — MCP SDK is a real dep, must be installed once
  • Two terminals open in the order-service repo, side-by-side if possible
  • Terminal 1 (leave it running): npm run mcp:http — starts the HTTP MCP server on http://localhost:3001/mcp
  • Terminal 2: register with Claude — claude mcp add --transport http order-store http://localhost:3001/mcp
  • Restart claude in Terminal 2 when prompted — the new tools only show up on a fresh session
  • Sanity-check: /mcp inside claude should list order-store as connected (green dot)
Paste this
Show me all our active orders. Which one is the highest monthly rate?
Expected: an mcp__order-store__* tool call, not a Read
● Calling MCP tool mcp__order-store__list_orders(status: "active") ● Result (5 active orders): ✔ ORD-0012 Globex Corp $4,500/mo ✔ ORD-0023 Initech LLC $1,200/mo ✔ ORD-0055 Umbrella Ltd $1,000/mo ✔ ORD-0067 Wonka Enterprises $4,500/mo ✔ ORD-0089 Stark Solutions $9,000/mo ● Five active orders. The highest monthly rate is ORD-0089 (Stark Solutions) at $9,000/mo — Platform Enterprise Annual.
That's not a grep. Not a file read. Claude called your API directly — same wire format as every other MCP server. Two commands of setup, localhost HTTP, and now Claude has eyes on your order store. The audit trail is on your server, not in your terminal scrollback.
Why HTTP and not stdio? On a locked-down corporate laptop, stdio can't always spawn a child process. Localhost HTTP slides through the allowlist — same MCP protocol, different transport. If you want to show the stdio path on a personal machine, the repo also ships npm run mcp + claude mcp add order-store -- node mcp/order-store/server.js.

Recovery: If Claude does a Read on a config file instead of calling the tool — interrupt: "Use the MCP tool." If /mcp shows the server as red/disconnected, check that Terminal 1 is still running and the port isn't already in use (lsof -iTCP:3001 on Mac/Linux, netstat -ano | findstr 3001 on Windows).
Ctrl-C Terminal 1 to stop the server. claude mcp remove order-store if cleaning up between sessions.

General presenter tips

  • Know the numbers. ORD-0042 drifts $67.50 (1.5%). ORD-0118 drifts $100.00 (3.7%). The green boxes in this doc are your ground truth — scan them before you present.
  • Paste, don't type. Every prompt in this doc is click-to-select-all. Fumbled typing kills momentum.
  • Narrate the wait. When Claude is reading, say what it's reading. "It's looking at the test output… now it's opening pricing.js… there's the Math.ceil."
  • Self-correction is a feature. If Demo 1 takes two passes instead of one — that's better. One-shot is a magic trick; two-shot is an engineering process the room can trust.
  • Time check: Demo 4 is skippable. If running long, skip Parallel Explore — it's read-only, no reset needed. Never cut 1, 3, or 6.
  • ORD-0042 is the through-line. Demos 1, 3, and 7 all touch the same order. Call it out: "Same order we fixed in Demo 1 — now we're reconciling it."
