## Professor

[C4E-exclusive.]

Here's the key insight about Claude Code Web: removing the terminal as a prerequisite is what unlocks the agent for everyone who isn't an engineer.

Notice what stays the same. Same agent — read, edit, run — same tool surface against your repos. The CLAUDE.md you wrote is loaded automatically because it's in the repo, not on someone's laptop. The skills you packaged into a plugin are available for the same reason. The substrate is identical.

What changes is who can reach it. PMs prototype without an eng handoff. Designers tweak CSS in the real code, not in a mockup. QA writes test cases directly against source. The terminal was the gating constraint — not the agent, not the skills, not the model. Removing it expands the audience by an order of magnitude.

Then the enterprise plumbing: SSO, audit logs, role-based access, central repo permissions. That's not incidental — those are what let admins grant non-engineers access without losing oversight. CC Web is the surface where Claude meets the rest of your organization.
