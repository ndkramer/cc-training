## Professor

Here's the key insight about Routines — renamed in 4.7, same idea: cron for Claude agents on Anthropic infrastructure.

Notice the connective tissue across the four modes. Ctrl+B is fire-and-forget — launch a one-off and walk away. /loop re-runs the same prompt on an interval. Monitor streams and reacts per line, event-driven. Routines runs without you, on Anthropic infrastructure, on a schedule you set.

The reason the pill row matters is that it grounds Routines in concrete work teams actually want done. Code health: nightly dep bumps, dead-code sweeps — work that gets postponed forever because no one's pager covers it. Security: CVE patches, secret scans — work that's important and tedious. Team glue: Monday changelog, stale-PR nudges — work that holds the team together but never makes it into a sprint.

The implication is that you're not automating a script. You're delegating a class of work — the work no one wants to own — to an agent that runs while you sleep.
